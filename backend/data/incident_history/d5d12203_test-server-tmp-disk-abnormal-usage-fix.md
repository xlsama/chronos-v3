# 事件排查报告

## 事件概要
- 标题：服务器 test-server /tmp 目录磁盘空间异常占用排查
- 严重程度：P3
- 处理状态：已解决

## 问题描述
- **受影响对象**：服务器 test-server（系统内识别为 test-target）
- **现象**：用户反馈磁盘使用率过高，具体表现为 `/tmp` 目录占用异常。
- **影响范围**：单台服务器存储空间及性能风险。

## 排查过程
1. **服务器确认**：调用 `list_servers()` 工具，确认目标服务器 ID 及主机名（host: localhost）。
2. **基础磁盘检测**：执行 `df -h` 命令。结果显示根分区（overlay）使用率为 4%（19G/592G），总体正常，需进一步排查局部目录。
3. **大文件定位**：执行 `du -sh /tmp/*` 并排序，发现 `/tmp/testfill` 文件占用 450M。
4. **文件属性核查**：执行 `ls -la /tmp/testfill`。
   - 文件类型：普通文件（-rw-r--r--）
   - 文件大小：471,859,200 字节（约 450MB）
   - 属主：root
   - 时间戳：Mar 17 07:49
5. **内容特征分析**：
   - 尝试 `file` 和 `xxd` 命令失败（环境缺少对应工具）。
   - 采用 `head -c 100 /tmp/testfill` 读取文件头，输出显示为连续的空字符（`\u0000`），判定为全零填充数据。

## 根因分析
- **根本原因**：`/tmp` 目录下存在一个名为 `testfill` 的非业务文件。
- **证据支撑**：
  1. `du` 命令明确显示该单一文件占用 450M。
  2. `head` 命令读取内容确认为无意义的零值填充（Padding Data），不具备任何业务功能。
  3. 该文件属于临时性或测试产生的残留数据，未及时清理导致了 `/tmp` 目录空间异常占用。

## 修复措施
1. **清理操作**：执行删除命令 `rm /tmp/testfill`，移除异常占用空间的文件。
2. **效果验证**：
   - **目录清理验证**：再次执行 `du -sh /tmp/*`，无返回结果，确认 `/tmp` 目录下无大文件残留。
   - **空间释放验证**：再次执行 `df -h`，结合 `du` 结果确认系统已释放相关存储空间（Avail 容量在统计上有所恢复）。
3. **后续建议**（基于 Agent 分析结论）：
   - 建议配置 `/tmp` 目录大小监控，设置阈值告警。
   - 考虑部署定期清理脚本或 Cron 任务，避免类似临时文件堆积。
